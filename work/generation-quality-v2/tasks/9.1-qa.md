# Task 9.1: QA — Acceptance Testing

## Description

Verify all 7 acceptance criteria from user-spec. Run through full generation flow with multiple templates, frameworks, edge cases. Test preserve-text mode through formatter. Test V1 endpoint.

## Acceptance Criteria (from user-spec)

1. [ ] Generation uses 3-agent pipeline (strategist -> copywriter -> formatter)
2. [ ] User can select a carousel framework before generation
3. [ ] Hook quality uses concrete formulas (not abstract labels)
4. [ ] Formatter prevents crashes from malformed AI output
5. [ ] All 23 existing templates continue to work
6. [ ] Generation cost stays within 3x of current per-carousel
7. [ ] Generation completes < 15s (p50)

## Test Scenarios

- Generate with each of the 6 frameworks
- Generate without framework (AI chooses)
- Generate with preserve-text mode
- Generate with all 4 content tones
- Generate with ToV guidelines
- Generate with brief
- Generate with 3, 5, 7, 9, 12 slides
- Test V1 B2B endpoint with API key
- Edge cases: very long text (3000 chars), special characters, unicode
- Error scenarios: invalid framework value, missing required fields
