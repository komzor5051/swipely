# Task 4.2: Refactor V1 B2B endpoint to use pipeline

## Description

Replace duplicated prompt logic in `/api/v1/generate/route.ts` with shared pipeline call. Keep B2B-specific auth (API key), quota management, response format, and optional PNG rendering. Preserve V1-specific presets as extension.

## Why

V1 endpoint currently duplicates all generation logic with its own copies of designPresets, buildSystemPrompt, etc. Using the shared pipeline eliminates duplication and gives B2B clients the same quality improvement from the multi-agent pipeline (D2, D8).

## Acceptance Criteria

- [ ] `v1/route.ts` calls `generateCarousel()` from shared pipeline
- [ ] All local copies of `designPresets`, `buildSystemPrompt`, `buildSlideStructure`, `cleanMarkdown`, `containsInjection` removed
- [ ] V1-specific preset `client_custom_v1` preserved as extension on shared presets
- [ ] API key auth (`Authorization: Bearer swp_live_*`) unchanged
- [ ] Quota management (`claimApiKeySlot`) unchanged
- [ ] Response format unchanged: `{ generation_id, slides, post_caption, view_url, edit_url }`
- [ ] Optional PNG rendering (`renderAndUploadSlides`) unchanged
- [ ] `framework` field optionally accepted in V1 API request
- [ ] `npm run build` passes

## Verify-smoke

`curl -X POST http://localhost:3000/api/v1/generate -H "Authorization: Bearer swp_live_test"` with test payload, verify response includes `generation_id` and valid slides.
