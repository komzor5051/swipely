# Task 2.3: Implement formatter agent

## Description

Create the formatter agent that validates and normalizes copywriter output into strict JSON matching the renderer's SlideData schema. Uses Gemini `responseSchema` for structural guarantees. Enforces word count, layout variety, element correctness, and clean text. Also handles preserve-text mode input.

## Why

The formatter is the safety net that prevents generation crashes (AC-4). Currently, if Gemini returns malformed JSON, wrong layouts, or exceeds word limits, the generation fails with a 500 error. The formatter ensures structurally valid output regardless of copywriter quality (D4, D11).

## Acceptance Criteria

- [ ] `lib/generation/agents/formatter.ts` exports `runFormatter(input: FormatterInput): Promise<FormatterOutput>`
- [ ] `lib/generation/prompts/formatter.ts` exports the formatter system prompt
- [ ] Uses Gemini `responseSchema` matching the exact SlideData[] structure from `components/slides/types.ts` (D4)
- [ ] Enforces `max_words_per_slide` per template from designPresets
- [ ] Enforces layout variety: min 3 different layout types for 5+ slide carousels (D11)
- [ ] Validates element structures match their type (e.g., `bar_chart` has `items[]` with label+value)
- [ ] Strips markdown artifacts, emoji, quotes from title and content
- [ ] Handles preserve-text mode: structures existing text into slides without rewriting (D5)
- [ ] Copywriter output is sanitized before injection into formatter prompt (D9)
- [ ] Output includes validated `slides: SlideData[]` and `postCaption: string`

## Verify-smoke

Feed intentionally malformed input (missing layouts, excess words, invalid element types) to formatter, verify it produces valid SlideData[] that passes type checks.
