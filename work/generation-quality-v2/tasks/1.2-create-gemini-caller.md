# Task 1.2: Create shared Gemini API caller

## Description

Extract Gemini fetch logic (URL construction, headers, timeout, retry on 503/429, JSON extraction, response parsing) into `lib/generation/gemini.ts`. Single `callGemini(prompt, config)` function that handles all error cases. Both endpoints use this instead of inline fetch.

## Why

Gemini call logic is duplicated in both endpoints with slightly different error handling. The multi-agent pipeline needs a reliable, reusable caller for 3 sequential agent calls. Centralizing also ensures consistent retry logic and timeout handling (D6).

## Acceptance Criteria

- [ ] `lib/generation/gemini.ts` exports `callGemini(prompt: string, config: GeminiConfig): Promise<GeminiResponse>`
- [ ] `GeminiConfig` includes: model, maxOutputTokens, temperature, responseSchema (optional), thinkingBudget
- [ ] Retry logic for HTTP 503/429 with configurable delay
- [ ] Timeout per call (configurable, default 20s per agent)
- [ ] JSON extraction from response with fallback regex for markdown-wrapped responses
- [ ] Clean error messages identifying failure stage (for pipeline error propagation, D10)
- [ ] Both endpoints refactored to use `callGemini()` instead of inline fetch
- [ ] Gemini proxy URL (`GEMINI_PROXY_URL`) support preserved
- [ ] `npm run build` passes
