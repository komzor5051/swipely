# Task 1.1: Extract generation utilities to lib/generation/

## Description

Extract `designPresets`, `contentTones`, `buildSlideStructure()`, `cleanMarkdown()`, `containsInjection()` from `route.ts` into dedicated files under `lib/generation/`. Create `presets.ts`, `slide-structure.ts`, `types.ts`. Update both endpoints to import from shared module. Sync `max_words_per_slide` values between presets and registry. Account for V1-specific presets (`client_custom_v1`).

## Why

Currently these are duplicated across `/api/generate/route.ts` and `/api/v1/generate/route.ts` with inconsistencies (e.g. `max_words_per_slide` differs between `designPresets` and `registry.ts`). Extracting to a shared module eliminates duplication and creates a single source of truth needed by the multi-agent pipeline (D2, D8).

## Acceptance Criteria

- [ ] `lib/generation/presets.ts` exports `designPresets` and `contentTones` with correct `max_words_per_slide` values synced with `registry.ts`
- [ ] `lib/generation/slide-structure.ts` exports `buildSlideStructure(count: number): string`
- [ ] `lib/generation/types.ts` exports `FrameworkId`, `StrategyOutput`, `CopywriterOutput`, `PipelineInput`, `PipelineOutput` types
- [ ] `lib/ai-utils.ts` remains the home for `cleanMarkdown()` and `containsInjection()` (already shared, just verify imports)
- [ ] Both `app/api/generate/route.ts` and `app/api/v1/generate/route.ts` import from shared module instead of local definitions
- [ ] V1-specific preset `client_custom_v1` preserved as V1 extension
- [ ] `lib/templates/registry.ts` imports `max_words_per_slide` from `presets.ts` (or values are synced)
- [ ] No functional changes to generation output — purely structural refactor
- [ ] `npm run build` passes
