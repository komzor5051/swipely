# Task 2.4: Create pipeline orchestrator

## Description

Create the pipeline orchestrator that runs the three agents sequentially. Handles framework selection routing, preserve-text mode (formatter only), error propagation with structured error messages per stage, and timeout management.

## Why

The orchestrator is the central coordination point for the multi-agent pipeline. It must handle the happy path (3 sequential calls), the preserve-text shortcut (formatter only), and all failure modes with clear error attribution (D10).

## Acceptance Criteria

- [ ] `lib/generation/pipeline.ts` exports `generateCarousel(input: PipelineInput): Promise<PipelineOutput>`
- [ ] Standard mode: runs strategist -> copywriter -> formatter sequentially
- [ ] Preserve-text mode: runs formatter only (D5)
- [ ] Framework passed through to strategist when provided
- [ ] Error propagation: when any agent fails, returns structured error with stage name and reason (D10)
- [ ] No silent fallback to old single-call path
- [ ] Timeout: 60s total, proportional per agent (~15s strategist, ~25s copywriter, ~20s formatter)
- [ ] Inter-agent output sanitization between each stage (D9)
- [ ] Passes `PipelineInput` fields to appropriate agents (ToV to copywriter, template constraints to formatter, etc.)
- [ ] Returns `PipelineOutput` with validated `slides` and `postCaption`

## Verify-smoke

Call `generateCarousel()` with test input (topic: "5 mistakes in Instagram marketing", template: "swipely", slideCount: 7), verify valid SlideData[] output with 7 slides.
