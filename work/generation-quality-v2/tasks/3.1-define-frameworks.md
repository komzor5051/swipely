# Task 3.1: Define carousel frameworks and hook library

## Description

Create framework definitions and hook library. Six frameworks: "mistakes", "case-study", "step-by-step", "before-after", "myths-vs-reality", "checklist". Each defines slide progression, hook formulas (2-3 per framework), CTA type, and optimal slide count range. Hook library contains 15-20 concrete fill-in-the-blank templates.

## Why

Current generation has no structured frameworks — AI decides structure freely, producing inconsistent results. Research shows top-performing carousels follow specific patterns. Frameworks encode these patterns for the strategist agent to use (D3, D7).

## Acceptance Criteria

- [ ] `lib/generation/frameworks.ts` exports `FRAMEWORKS: Record<FrameworkId, Framework>` and `HOOK_LIBRARY: HookTemplate[]`
- [ ] 6 frameworks defined with Russian display labels:
  - `mistakes`: "3 ошибки" — mistake callout, slide progression: hook -> mistake 1-3 -> solution -> CTA
  - `case-study`: "Кейс-разбор" — results breakdown with before/after data
  - `step-by-step`: "Пошаговый гайд" — sequential steps with clear numbering
  - `before-after`: "До/После" — transformation story with contrast
  - `myths-vs-reality`: "Мифы vs Реальность" — stereotype breaking
  - `checklist`: "Чек-лист" — actionable list with checkmarks
- [ ] Each framework defines: `slideProgression` (array of slide types), `hookFormulas` (2-3 concrete templates), `ctaType`, `optimalSlideRange` (min/max)
- [ ] Hook library has 15-20 templates as fill-in-the-blank patterns (e.g., "{N} mistakes that cost you {outcome}")
- [ ] Each hook template categorized by applicable frameworks
- [ ] Types exported from `lib/generation/types.ts` (Framework, HookTemplate interfaces)
