**Step 1: Extend types.ts**

Add the following types AFTER the existing SlideLayout type and BEFORE the SlideData interface:

```typescript
// Rich element data types
export interface ChartItem {
  label: string;
  value: number;
}

export interface ListElementData {
  type: "list";
  items: ChartItem[]; // 3-7 items; only label is used for display, value is ignored (unified schema with charts for Gemini compatibility)
}

export interface StatElementData {
  type: "stat";
  value: string;  // e.g. "87%", "2.4M", "$150K"
  label: string;  // supporting description
}

export interface BarChartElementData {
  type: "bar_chart";
  items: ChartItem[]; // 3-5 bars
}

export interface PieChartElementData {
  type: "pie_chart";
  items: ChartItem[]; // 3-5 slices, raw numbers (component computes %)
}

export type SlideElement = ListElementData | StatElementData | BarChartElementData | PieChartElementData;
```

Then add to SlideData interface:
```typescript
element?: SlideElement; // Rich element — at most one per slide
```

IMPORTANT: `ListElementData.items` uses `ChartItem[]` (not `string[]`) because the Gemini responseSchema uses a unified `{label, value}` shape for ALL element types — Gemini structured output does not support discriminated unions. For list elements, the AI puts list item text in `label` and sets `value` to 0. `ListElement` component reads only `item.label`.

- Max 5 bars. Calculate max value from items, scale bar heights proportionally.

**PieChartElement.tsx** — Inline SVG pie/donut chart:
- Props: `{ items: ChartItem[]; accentColor: string }`
- Max 5 slices. Component computes percentages from raw `value` numbers.
- Color palette: Use accentColor at 5 opacity steps: `[1.0, 0.75, 0.55, 0.40, 0.28]`. Convert accentColor hex to rgba with these opacities. Helper function `hexToRgba(hex: string, opacity: number): string` inside the file.
- SVG viewBox: `"0 0 400 400"`. Donut chart centered at (200, 200), outer radius 160, inner radius 90 (donut hole).
- Each slice: `<path>` using arc segments. Calculate start/end angles from cumulative percentages. Use standard SVG arc path formula.
- Inline labels: `<text>` positioned at the midpoint angle of each slice, at radius 130 (between inner and outer). Show `label` text. `fontSize="16"`, `fill="#FFFFFF"`, `textAnchor="middle"`, `dominantBaseline="middle"`.
- For small slices (<10%), place label outside at radius 190 with a thin line connector.
- Wrapper: `<svg>` element with `width="100%"`, `style={{ maxWidth: 400 }}`

**Step 3: Create elements/index.tsx — renderElement dispatcher**

```typescript
import React from "react";
import type { SlideElement } from "../types";
import { ListElement } from "./ListElement";
import { StatElement } from "./StatElement";
import { BarChartElement } from "./BarChartElement";
import { PieChartElement } from "./PieChartElement";

interface RenderElementProps {
  element: SlideElement;
  accentColor: string;
}

export function renderElement({ element, accentColor }: RenderElementProps): React.ReactNode {
  switch (element.type) {
    case "list":
      return <ListElement items={element.items} accentColor={accentColor} />;
    case "stat":
      return <StatElement value={element.value} label={element.label} accentColor={accentColor} />;
    case "bar_chart":
      return <BarChartElement items={element.items} accentColor={accentColor} />;
    case "pie_chart":
      return <PieChartElement items={element.items} accentColor={accentColor} />;
    default:
      return null;
  }
}
```
