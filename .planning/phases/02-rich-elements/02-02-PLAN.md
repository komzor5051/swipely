  <action>
For EVERY template file, apply the following pattern. Read each file before modifying to understand its specific content zone structure.

**Step 1: Add import**

At the top of each template file, add:
```typescript
import { renderElement } from "../elements";
```

**Step 2: Identify the template's ACCENT color constant**

Each template defines an accent color at file top (e.g., `const ACCENT = "#5B4FE8"` in DispatchSlide). Note the variable name — it varies per template.

**Step 3: Find the content zone and add element rendering**

In each template's JSX, find where `slide.content` is rendered (usually a `<p>` tag that calls `renderContent(slide.content)` or directly renders `{slide.content}`). This is the content zone.

Replace the content rendering with a conditional:

```tsx
{slide.element ? (
  <div style={{ marginBottom: 16 }}>
    {renderElement({ element: slide.element, accentColor: ACCENT })}
    {slide.content && (
      <p style={{
        fontSize: 22,
        color: "rgba(255,255,255,0.6)",
        marginTop: 12,
        fontFamily: /* use the template's existing content font */,
        lineHeight: 1.4,
      }}>
        {slide.content}
      </p>
    )}
  </div>
) : (
  /* ORIGINAL content rendering — keep exactly as-is */
  <p style={/* original styles */}>
    {renderContent(slide.content)}
  </p>
)}
```

CRITICAL rules for this step:
- The ORIGINAL content rendering (the `else` branch) MUST be kept EXACTLY as it was — same styles, same scaleContentFontSize call, same maxWidth, everything. This ensures zero visual regression for slides without elements.
- The `accentColor` passed to `renderElement` MUST use the template's own accent color constant (e.g., `ACCENT` in DispatchSlide, whatever the variable is named in each template).
- The caption `<p>` inside the element branch uses a smaller font (22px), muted color (rgba 0.6 opacity), and the template's own content font-family.
- If a template has MULTIPLE content zones (e.g., different layout branches like DispatchSlide's `hero` vs `content` branches), add the element conditional to EACH content zone.
- Keep the title rendering completely unchanged — `slide.title` and `renderTitle()` calls remain as-is.

**Templates with special handling:**

- **PhotoSlide**: This template is used for Photo Mode only. Rich elements are unlikely on photo slides, but still add the import and conditional for completeness. Use `"#FFFFFF"` as accentColor since PhotoSlide has no accent constant (it renders on photo backgrounds).

- **SwissSlide**: Has a two-column layout. The element should render in the right column (content column), not the left.

- **NikkeiSlide**: Has two branches (hook vs content). Add element conditional to both branches.

- **Templates with very custom layouts** (ReceiptSlide, SpeechBubbleSlide, GridMultiSlide): Read each one carefully. The content zone may not be a simple `<p>` tag. Find where `slide.content` is used and wrap that specific section.

Process all 24 templates. After all changes, run TypeScript check.
