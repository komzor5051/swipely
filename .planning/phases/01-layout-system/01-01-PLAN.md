# Plan 01-01: Extend SlideData schema with layout enum + update Gemini responseSchema

<objective>
Add an AI-assigned `layout` field to every slide so that Gemini picks a different visual composition per slide. This plan defines the data contract (types + API schema) without changing any template rendering.

Purpose: Foundation for Phase 1 -- without the schema and Gemini constraint, templates have no layout signal to render against.
Output: Updated SlideData type, updated utils, Gemini API route with responseSchema enum.
</objective>

The current `SlideData` type has only `type`, `title`, `content`, `imageUrl?`. Templates use a deterministic `getLayoutVariant()` that maps slide type to layout -- meaning all slides of the same type get the same layout. The goal is for Gemini to assign a `layout` value per slide from a fixed enum, ensuring visual variety.

Key constraint: Gemini REST API supports `responseSchema` in `generationConfig` to enforce structured JSON output. The `layout` field must be declared as an enum there so Gemini is forced to use valid values and vary them.

The new layout enum values: `"text-left" | "text-right" | "split" | "big-number" | "quote" | "default"`.

**Important:** This plan does NOT update template files to pass `slide.layout` to `getLayoutVariant`. That wiring happens in plan 01-02, which updates all 24 templates. Until 01-02 runs, templates continue using the fallback path (type-based layout), which is correct backward-compatible behavior.

     ```
       slideNumber: number,
       totalSlides: number,
       slideLayout?: SlideLayout
     ): LayoutVariant {
       // If AI assigned a layout, use it directly
       if (slideLayout && slideLayout !== "default") return slideLayout;
       // Fallback: existing deterministic logic
       if (slideNumber === 1) return "text-left";
       if (slideNumber === totalSlides) return "default";
       return FALLBACK_MAP[slideType] ?? "default";
     }
     ```
   - Update `getContentAlignment` to work with the new layout values:
     ```
     export function getContentAlignment(
       layout: LayoutVariant,
       slideNumber: number
     ): "flex-start" | "center" | "flex-end" | "space-between" {
       if (layout === "big-number" || layout === "quote") return "center";
       if (layout === "text-left") return "flex-start";
       if (layout === "text-right") return "flex-end";
       if (layout === "split") return slideNumber % 2 === 0 ? "flex-end" : "flex-start";
       // "default": alternate top/bottom
       return slideNumber % 2 === 0 ? "flex-start" : "flex-end";
     }
     ```
