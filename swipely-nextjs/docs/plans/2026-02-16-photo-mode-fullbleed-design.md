# Photo Mode: Full-Bleed Image Generation

**Date**: 2026-02-16
**Scope**: swipely-nextjs only
**Approach**: Full-bleed images + CSS overlay for text readability

## Problems

1. **Square images with black bars in portrait** — prompt tells model "leave 20% top and 25% bottom empty" → model generates content in center square
2. **Text on AI images** — no-text constraint not aggressive enough
3. **Generic scenes** — separate `generateSceneDescriptions()` call produces template poses

## Solution

### 1. `lib/services/image-generator.ts` — Prompt Overhaul

**Remove:**
- `generateSceneDescriptions()` function
- `SceneDescription` interface
- "empty space" requirements from `buildImagePrompt()`

**New `buildImagePrompt()` signature:**
```ts
buildImagePrompt(slide: { title: string; content: string; type: string; slideNumber: number }, style: ImageStyle, format: Format): string
```

**New prompt structure:**
- Slide context (title + content) embedded directly — Gemini chooses scene
- Full-bleed composition: "fill entire frame edge-to-edge"
- Soft compositional hint: "slightly darker tones in upper/lower thirds" (NOT empty space)
- Enhanced no-text block with negative prompting patterns

### 2. `app/api/generate/photo/route.ts` — Pipeline Simplification

- Remove `generateSceneDescriptions()` call (saves 1 API call + 1-2s)
- Pass `{title, content, type, slideNumber}` to `generateSlideImage()` directly

### 3. `PhotoSlide.tsx` — No Changes

Existing multi-layer gradient overlay + vignette + glass card already handles full-bleed images well.

## Expected Outcome

- Images fill entire canvas (4:5 or 1:1) without black bars
- Diverse scenes reflecting each slide's meaning
- No text generated by AI on images
- Faster generation (-1 API call)
